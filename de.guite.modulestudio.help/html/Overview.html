Introduction

ModuleStudio represents a generative architecture for the Zikula Application Framework. Based on principles of MDSD the creation of modules is being abstracted to a new level.

ModuleStudio uses frameworks and technologies introduced in Zikula and creates new ways of abstraction to describe modules. Therefore the boring writing of schematic code is not applicable, the development is focused on the module's functional differences compared to other modules.

ModuleStudio takes module development to a new level as it displaces the typing of source code by a graphical modeling process. All modules generated by ModuleStudio are automatically API-compliant and match the actual conventions relating security and usability. When new Core versions occur all changes are updated so all modules will be ready for a new release after simply pressing a button. ModuleStudio therefore reduces architectural motivated adjustments and increases maintainability as well as reusability of Zikula's common artefacts.
